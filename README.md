# Task Manager App (Flutter + Firebase)

A task management app with email/password authentication, per-user task
storage in Firestore, and local due-date reminder notifications.

## Features

- **Home screen**: lists tasks with title, status, and due date
- **Filters**: All / Completed / Pending (chips at the top of Home)
- **Add / Edit / Delete tasks**, each with Title, Description, Due Date, Priority
- **Mark complete** via checkbox (swipe left on a task to delete)
- **Firebase Authentication** (email/password) for register/login
- **Firestore** storage, scoped per user via security rules
- **Due-date reminders** via local notifications (1 day before + at due time)
- **State management**: Provider (`AuthProvider`, `TaskProvider`)

## Setup

### 1. Prerequisites

- Flutter (latest stable) — https://docs.flutter.dev/get-started/install
- A Firebase project — https://console.firebase.google.com
- Dart's FlutterFire CLI: `dart pub global activate flutterfire_cli`
- Firebase CLI: `npm install -g firebase-tools`, then `firebase login`

### 2. Scaffold native platform folders

This deliverable contains only the `lib/` Dart source plus `pubspec.yaml` —
no Android/iOS/web platform folders, since those are normally generated by
the Flutter tool itself and tied to a specific Flutter SDK version. From the
project root:

```bash
flutter create . --org com.yourcompany --project-name task_manager_app
```

This will scaffold `android/`, `ios/`, `web/`, etc. without touching your
existing `lib/` files.

### 3. Connect Firebase

```bash
flutterfire configure
```

Follow the prompts to select (or create) your Firebase project and the
platforms you want (Android/iOS/web/macOS). This **overwrites**
`lib/firebase_options.dart` with real values — the version in this
deliverable is a placeholder and will not work as-is.

### 4. Enable Email/Password authentication

In the Firebase console: **Authentication → Sign-in method → Email/Password
→ Enable**.

### 5. Create Firestore database

In the Firebase console: **Firestore Database → Create database**. Choose
production mode (rules are provided in `firestore.rules`).

Deploy the included security rules:

```bash
firebase init firestore   # point it at this project, accept defaults
firebase deploy --only firestore:rules
```

The rules ensure a signed-in user can only read/write task documents whose
`userId` field matches their own `uid` — so tasks are private per account.

### 6. Install dependencies

```bash
flutter pub get
```

### 7. Notification permissions

- **Android 13+**: the app requests the `POST_NOTIFICATIONS` runtime
  permission on first launch (handled in `NotificationService.init()`).
  Add this to `android/app/src/main/AndroidManifest.xml` if not already
  present after `flutter create`:
  ```xml
  <uses-permission android:name="android.permission.POST_NOTIFICATIONS"/>
  <uses-permission android:name="android.permission.SCHEDULE_EXACT_ALARM"/>
  ```
- **iOS**: notification permission is requested automatically via
  `DarwinInitializationSettings`. Ensure "Push Notifications" and
  "Background Modes → Background fetch" capabilities are enabled in Xcode
  if you extend this to remote push later.

### 8. Run the app

```bash
flutter run
```

## About the notification approach

The app schedules **local** notifications (via `flutter_local_notifications`)
1 day before and at the moment a task is due. This works entirely on-device,
needs no backend, and satisfies "notify the user as a task nears its due
date" without extra infrastructure.

If you need true **server-triggered push** (e.g., reminders that still
arrive if the user force-quit the app weeks ago, or cross-device sync), pair
this with a Firebase Cloud Function + FCM. Sketch:

```js
// functions/index.js (Cloud Functions, Node.js)
const functions = require("firebase-functions");
const admin = require("firebase-admin");
admin.initializeApp();

// Scheduled function, e.g. runs every 15 minutes via Cloud Scheduler,
// checks for tasks due soon and sends an FCM push to the owning user's
// registered device token.
exports.notifyUpcomingTasks = functions.pubsub
  .schedule("every 15 minutes")
  .onRun(async () => {
    const now = admin.firestore.Timestamp.now();
    const soon = admin.firestore.Timestamp.fromMillis(
      now.toMillis() + 60 * 60 * 1000
    );
    const snap = await admin
      .firestore()
      .collection("tasks")
      .where("isCompleted", "==", false)
      .where("dueDate", ">=", now)
      .where("dueDate", "<=", soon)
      .get();

    // For each task, look up the user's stored FCM token and send a push.
    // (Requires storing a `fcmToken` field on a `users/{uid}` doc.)
  });
```

This is optional — the local-notification approach in this repo is
sufficient for the stated requirement and avoids paid Cloud Functions usage.

## Notes / things to double check before shipping

- Dependency versions in `pubspec.yaml` were current as of early 2026;
  run `flutter pub outdated` and bump as needed.
- The placeholder `firebase_options.dart` **must** be regenerated via
  `flutterfire configure` — the app will crash on `Firebase.initializeApp()`
  otherwise.
- Firestore composite query (`where userId == ... orderBy dueDate`) may
  prompt Firebase to ask you to create a composite index the first time you
  run it — just click the link it prints in the console/logs.
"# task_manager_app"
